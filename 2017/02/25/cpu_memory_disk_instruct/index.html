cpu、内存、硬盘、指令之间的关系

Posted on 2017-02-25   |   In 计算机体系结构   |  

简单的介绍了cpu、内存、硬盘、指令之间的关系

1. cpu

cpu是一台计算机中的中央处理单元, 它用于执行计算机程序的指令, 像执行基本的算术操作, 逻辑操作, 输入输出操作等. cpu读取指令, 然后根据指令的内容操作计算机的其他部件完成指令所代表的任务. cpu就像人类的大脑控制着其他的部件完成任务.

2. 内存

内存是cpu读取指令的地方, 当cpu执行任务时, 根据pc寄存器中的内容(pc寄存器存储的是下一条指令在内存中的地址)找到指令所在的位置, 然后根据指令的内容完成相应的操作. 内存一般是DRAM(Dynamic Random Access Memory), 也就是说内存不能长时间的保存内容, 只要一断电, 内存中的内容就消失了.因为现在cpu的速度和内存的速度相差较大, 所以在内存和cpu之间还会有缓存的存在, 根据局部性原理, 内存的一部分内容会暂存在缓存中, 以提高cpu的读取速度.

3. 硬盘

硬盘也是计算机中的存储部件, 所有有用的内容正常情况下都会被保存到硬盘中, 因为硬盘可以长时间保存内容, 即使断电也不会消失. 但是cpu并不能直接读取硬盘中的内容, cpu必须根据指令控制I/O部件才能读取硬盘中的内容, 硬盘中的程序也必须读到内存中才能被cpu执行.

4. 指令

指令是控制cpu执行的命令. 不同cpu平台上的指令有所不同, 例如ARM的cpu和Intel的cpu支持的指令就不相同. 指令中的操作码告诉cpu要进行什么操作, 例如是加法操作还是赋值操作, 指令中的操作数告诉cpu这次的操作是针对哪些数据进行操作的. 因此我们可以通过指令控制cpu来完成我们想让它完成的任务.
