2. ApplicationContext 和 WebApplicationContext

  • 区别: ServletContext 是web应用程序的大环境,用于存储整个web应用程序级别的对象,由spring管理的对象的Map结构就是存储在ServletContext中的.
    ApplicationContext,WebApplicationContext 是Spring的BeanFactory,一个是支持web特性的BeanFactory.

  • Spring获取ApplicationContext和WebApplication的方法

    1. 在初始化时保存ApplicationContext对象

    2. 通过Spring提供的工具类获取ApplicationContext对象

      说明: 这种方式适合于采用Spring框架的B/S系统,通过ServletContext对象获取ApplicationContext对象,然后在通过它获取需要的类实例.
      上面两个工具方式的区别是,前者在获取失败时抛出异常,后者返回null.
      其中 servletContext sc 可以具体 换成 servlet.getServletContext()或者 this.getServletContext() 或者 request.getSession().getServletContext(); 另外，由于spring将注入的对象放在ServletContext中的，所以可以直接在ServletContext取出 WebApplicationContext 对象:
      WebApplicationContext webApplicationContext = (WebApplicationContext) servletContext.getAttribute(WebApplicationContext.ROOT_WEB_APPLICATION_CONTEXT_ATTRIBUTE);

    3. 继承自抽象类ApplicationObjectSupport
      说明: 抽象类ApplicationObjectSupport提供getApplicationContext()方法,可以方便的获取到ApplicationContext.
      Spring初始化时,会通过该抽象类的setApplicationContext(ApplicationContext context)方法将ApplicationContext对象注入.

    4. 继承自抽象类WebApplicationObjectSupport
      说明: 类似方法3,调用getWebApplicationContext()获取WebApplicationContext

    5. 实现接口ApplicationContextAware
      说明: 实现该接口的setApplicationContext(ApplicationContext context)方法,并保存ApplicationContext 对象.Spring初始化时，会通过该方法将ApplicationContext对象注入.

    参考: ApplicationContext和WebApplicationContext

3. DispatcherServlet的默认配置

  • DispatcherServlet的默认配置在DispatcherServlet.properties定义, 在org.springframework.web.servlet包中

###　4. DispatcherServlet处理请求的顺序

  • WebApplicationContext作为一个属性将被绑定到请求中, 相应的属性名:DispatcherServlet.WEB_APPLICATION_CONTEXT_ATTRIBUTE, 这样在整个请求过程中的Controller和其他元素就可以使用WebApplicationContext
  • The locale resolver(用于国际化)被绑定到请求过程中, 可以不需要
  • The theme resolver(用于应用的主题)被绑定到请求过程中, 可以不需要
  • 如果指定了multipart file resolver且请求中包含multiparts,请求将被包在 MultipartHttpServletRequest中
  • 需找处理这次请求的合适方法, 如果找到了则调用执行链进行处理(preprocessors -> controllers -> postprocessors)生成待渲染的模型
  • 返回模型, 渲染视图, 也有可能没有模型返回, 因为有可能是preprocessor或者 postprocessor拦截了请求
  • Handler exception resolvers 用于处理请求出错时返回的视图
  • DispatcherServlet也支持Last-Modify, 只要Controller方法实现LastModify接口实现getLastModified方法就可以了, 具体参考http://www.iteye.com/topic/1121788
