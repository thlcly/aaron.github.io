lombok使用笔记

Posted on 2016-08-29   |   In Node   |  

This is an application notes for Lombok

1. @NotNull

  • 概述

@NotNull 可以用在方法或者构造函数的参数上, 用于生成非空校验语句
生成的校验语句类似于if (param == null) throw new NullPointerException("param");, 同时校验语句会加入到方法的最开始的位置, 如果是构造函数则是出现在super()或者this()语句之后
如果非空校验语句已经在方法顶部生成, 则@NotNull不会再生成校验语句

2. @Cleanup

  • 概述

@Cleanup确保在代码执行完成之后指定的资源会被安全的释放, @Cleanup默认调用的是资源的close的方法, 如果资源不是调用close方法进行释放则在@Cleanup("method name")中指定, @Cleanup只支持无参的方法, 默认调用的是close方法. 只有在资源不为null时才会调用close方法. 如果被@Cleanup包围的代码中抛出异常, 则这个异常将被lombok隐藏, 而抛出clean up的异常, 这点要注意

3. @Getter / @Setter

  • 概述

@Getter/@Setter用于生成get/set方法, 默认的getter/setter方法生成的方法是get+字段名/set+字段名(参数类型 参数名), 如果字段类型是boolean, 则是is+字段名/is+字段名(参数类型 参数名). 生成的方法默认是public的, 可以通过AccessLevel.xxx进行修改, 如果AccessLevel.NONE, 则默认不生成get/set方法. @Getter/@Getter只会为非静态字段生成get/set方法. 可以使用onMethod＝@(), 在生成的方法上添加注解, 可以使用onParam=@(), 在生成的方法的参数上添加注解. @Getter/@Setter既可以用在字段上也可以用在类上, 如果用在类上则默认为类中所有的非静态字段生成get/set方法

4. @ToString

  • 概述

@ToString用于生成toString()方法, 默认情况下生成的toString()方法会打印类名, 按声明顺序的逗号隔开的字段. 默认情况下所有非静态的字段将被打印. 可以使用exclude指定不被打印的字段, 使用of指定要被打印的字段. callSuper为true则会打印父类的toString()方法. includeFieldNames为true则会带上字段名. doNotUseGetters为true且字段的get方法存在则会调用字段的get方法而不是直接使用字段的引用.
