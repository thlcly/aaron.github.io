Thymeleaf中遇到的坑总结

Posted on 2016-08-04   |   In Thymeleaf   |  

This is experience from using thymeleaf in actual project

1. Thymeleaf 2中关闭标签的强校验

1.1 Tymeleaf标签强校验引出的问题

  • 每一个html标签必须闭合
  • 如果页面中存在js代码, 则js代码不能出现(<,>…)等一些操作符, 因为这个标签和xml的标签存在冲突
  • 如果使用了前端模版, 则模版中也不能出现上述的一些标签

1.2 对于js代码的转义

如果只是js代码中要包含一些操作符而导致thymeleaf页面不能编译通过, 可以使用xml注释语法进行过滤

1.3 如果是前端模版中存在不能被thymeleaf解析的内容, 那xml的注释方式就无能为力了, 只能关闭thymleaf的标签强校验了

思路: 查看thymeleaf的官方文档中提到, thymeleaf支持多种模式去解析页面, 大致可以分成两类, 一类是使用XML形式的(XML, Valid XML, XHTML, Valid XHTML, HTML5), 一类是不使用XML形式的(Legacy HTML5), 所以要关闭标签的强校验应该将thymeleaf的模式配置成Legacy HTML5的模式(以下使用spring boot工程进行讲解)

  • 在spring boot中配置tymeleaf使用的是org.springframework.boot.autoconfigure.thymeleaf.ThymeleafAutoConfiguration这个class, 所以要修改thymeleaf的默认的校验模式也从这个类下手

  • ThymeleafAutoConfiguration部分源码

  • ThymeleafProperties部分源码

    所以要修改thymeleaf的默认模式, 就在application.properties中配置spring.thymeleaf.mode=LEGACYHTML5

  • 配置完成mode之后启动程序发现thymeleaf模版报错, 错误很明显, 就是少了一个jar, 将缺少的jar放入classpath中即可
