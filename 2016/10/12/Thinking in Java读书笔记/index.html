1. Javadoc

1.1 独立文档

以”@”字符开头的命令, 置于注释行的最前面(不算前导”*”之后的最前面)

1.2 行内文档标签

可以出线在javadoc注释中的任何地方, 也是以”@”开头, 但是要在花括号内

1.3 注释文档的类型

  • 类: 正好位于类定义之前
  • 域(类的属性): 正好位于域定义之前
  • 方法: 正好位于方法定义之前

注意点: javadoc只能为public和protected成员进行文档注释. private和包内可访问成员的注释会被注释掉, 但是可以用-private进行标记(可以把private成员的注释也包括在内). 这样做的原因是因为只有public和protected成员才能在文件之外被使用, 所以也只有这两种访问级别有注释的需要

1.5 标签示例

  • @see: 引用其它类

    javadoc会在生成的文档中加入一个具有超链接的”See Also”

    eg:

  • {@link package.class#member label}

    该标签作用于行内, 并且是用”label”作为超链接文本

  • {@docRoot}

    产生到文档根目录的相对路径

  • {@inheritDoc}

    从当前这个类的最直接基类中继承相关文档到当前文档注释中

  • @version

    该标签的格式: @version version-information
    可以使用javadoc -version从生成的文档中提取出版本的信息

  • @author

    该标签的格式: @author author-information
    可以使用javadoc -author从生成的文档中提取出版本的信息

  • @since

    用于指定程序代码最早开始引入的版本

  • @param

    该标签的格式: @param parameter-name description
    parameter-name是方法参数列表中的标识符, description是对参数的描述

  • @return

    该标签的格式: @return description
    用description来描述返回值的含义

  • @throws

    该标签的格式: @throws fully-qualified-class-name description
    用于描述该方法可能会抛出的异常

  • @Deprecated

    用于表示被该标签注解的元素已经过时, 不再建议使用, 该元素在将来的版本中将会被删除
